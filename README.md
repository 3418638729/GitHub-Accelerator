## 🚀 快速使用

### 1️⃣ 下载脚本

方法 A：直接下载本仓库的 `GitHub.bat`

方法 B：克隆仓库
```bash
git clone https://github.com/3418638729/GitHub-Accelerator.git
```

### 2️⃣ 双击运行

找到 `GitHub.bat`，**双击**即可：
- 自动弹出浏览器打开 GitHub
- 自动请求管理员权限（需要点"是"）
- 自动下载最新 hosts → 更新系统 hosts → 刷新 DNS

**全程自动，不用你点任何东西。**

### 3️⃣ （可选）安装开机自启

以管理员身份运行：
```cmd
GitHub.bat /install
```

之后每次开机登录，30 秒后脚本会在后台**静默运行**（无窗口、无弹窗），检查 hosts 是否需要更新。

## 📖 详细说明

### 适用场景

| 症状 | 说明 |
|------|------|
| 🐢 GitHub 加载慢 | 图片、CSS 加载半天 |
| 🚫 GitHub 打不开 | 浏览器显示"无法访问此网站" |
| 🖼️ 头像/图片裂开 | avatars.githubusercontent.com 无法连接 |
| 📦 Clone 失败 | `git clone` 速度极慢或超时 |
| 🔄 RAW 文件下不了 | raw.githubusercontent.com 无法访问 |

### 文件说明

| 文件 | 位置 | 说明 |
|------|------|------|
| `GitHub.bat` | 桌面（或任意位置） | ✅ 主程序，双击运行 |
| `gh520_launcher.vbs` | 桌面 | 开机静默启动器（/install 时自动生成） |
| 缓存文件 | `%TEMP%\gh520\` | 缓存最新 hosts，避免重复下载（自动管理） |

### 数据来源

本工具使用 [GitHub520](https://github.com/521xueweihan/GitHub520) 项目维护的最新 GitHub IP 列表，该项目持续更新，保证可用性。

## ⌨️ 命令大全

| 命令 | 说明 |
|------|------|
| **双击 `GitHub.bat`** | 🔥 最常用。打开 GitHub + 自动更新 hosts |
| `GitHub.bat /install` | 安装开机计划任务（静默更新） |
| `GitHub.bat /uninstall` | 卸载计划任务 + 清除缓存 |
| `GitHub.bat /check` | 检查当前状态（hosts / 缓存 / 任务） |
| `GitHub.bat /startup` | 静默更新模式（一般不用手动调用） |

### 计划任务（安装后）说明

```
名称：    GitHub-Hosts-Update
触发器：  用户登录时
延迟：    30 秒（避免开机卡顿）
权限：    最高（可修改 hosts）
窗口：    无（通过 VBS 静默启动）
行为：    检查缓存 → 今天已更新 → 0.3 秒退出
         检查缓存 → 今天未更新 → 下载最新 → 更新 hosts
```

## ❓ 常见问题

### Q: 双击后弹出了管理员窗口，然后呢？
正常！第一次运行需要管理员权限才能修改 hosts 文件，点"是"就好。这是 Windows 的安全机制。

### Q: 双击后浏览器没打开？
可能是 `start "" "https://github.com"` 被安全软件拦截了，检查一下浏览器或防火墙设置。

### Q: 浏览器打开了但 GitHub 还是打不开？
可能是你的网络环境（公司网络、校园网、某些地区）直接屏蔽了 GitHub IP。这种情况下 hosts 加速无效，需要使用代理/VPN。

可以运行 `GitHub.bat /check` 检查状态，确认 hosts 是否已更新。

### Q: 过了很久没用，还能正常用吗？
能。每次双击都会自动联网拉最新的 IP 列表，不管隔了多久。即使下载失败，也会使用上次缓存的 hosts。

### Q: 开机计划任务会卡吗？
不会。**缓存命中时仅需 ~0.3 秒**（检查缓存 + 刷新 DNS），后台静默运行，你完全感觉不到。

### Q: 如何卸载？

```cmd
GitHub.bat /uninstall
```

这会：
- 删除计划任务（不会再开机自启）
- 清除缓存文件
- 不修改 hosts 文件（已添加的内容保留）

如果想恢复原始 hosts，手动打开 `C:\Windows\System32\drivers\etc\hosts`，删除 `# GitHub520 Host Start` 到 `# GitHub520 Host End` 之间的内容即可。

### Q: 会不会影响其他网站？
不会。脚本只修改 GitHub 相关域名（github.com、raw.githubusercontent.com、avatars.githubusercontent.com 等），不影响其他网站。

## ⚙️ 运行原理

```text
用户双击 GitHub.bat
       │
       ▼
┌─────────────────────┐
│ 打开 GitHub 浏览器   │  ← 立即弹出，不卡
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 请求管理员权限       │  ← 需要写 hosts 文件
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 检查缓存是否今天     │
│                     │
│  是 ──→ 刷新 DNS    │  ← ~0.3 秒，秒退
│  否 ──→ 下载最新 IP │  ← 从 GitHub520 拉取
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 更新系统 hosts 文件  │  ← 替换旧的 IP 列表
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 刷新 DNS 缓存       │  ← 立即生效
└─────────────────────┘
```

### 三重故障保障

```
第 1 层：从 GitHub520 在线下载最新 IP 列表 ✅
   ↓ 失败（网络不通）
第 2 层：使用本地缓存的上次有效 hosts ✅
   ↓ 失败（缓存被删除）
第 3 层：使用脚本内置的备用 IP 列表 ✅（部分可能可用）
```
